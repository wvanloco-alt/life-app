# Spec: UI Refinements & Heatmap

## 1. Background & Objective
As the app has grown, certain UI elements have become cluttered or irrelevant. The Goals form has too many fields for a standard narrow modal, requiring excessive scrolling. Additionally, the Mission and Analytics tabs are not providing daily value, violating the "Underdo the competition" principle. Finally, the Weekly Activity Volume stacked bar chart is visually heavy and doesn't align with the "Consistency over intensity" principle. 

The objective of this spec is to refine the UI by widening the Goals form, removing unused features, and replacing the volume chart with a GitHub-style activity heatmap that rewards daily consistency.

## 2. Scope

### 2.1 Goals Form Redesign
- Expand the `DialogContent` width for the Goal creation and edit forms (e.g., `max-w-3xl` or `max-w-4xl`).
- Reorganize the layout into a two-column grid.
  - **Left Column**: Core goal information (Type/Horizon, Title, Roles, Target Value, Unit, Activity Type, Description).
  - **Right Column**: Scheduling Rules (Preferred Days, Preferred Time, Session Pattern) and Monthly Benchmark auto-generation toggle.

### 2.2 Feature Trimming (Mission & Analytics)
- Remove the `/mission` and `/analytics` routes and pages.
- Remove their links from the `app-sidebar.tsx` component.
- Remove their respective API endpoints (`/api/mission`, `/api/analytics`).
- Remove `MissionStatement` and `MissionStatementVersion` from `src/db/schema.ts` (this will require dropping the tables).

### 2.3 Activity Heatmap (GitHub Style)
- Remove the current "Weekly Activity Volume" stacked bar chart from the Activities Dashboard.
- Build a new `ActivityHeatmap` component.
- The heatmap should display a 52-week grid (similar to GitHub contributions), showing the last year of activity.
- Each day block should represent whether an activity was completed.
- Color coding should represent intensity or simply "active vs inactive" (e.g., green shades). Given the principle "Consistency over intensity," a binary active/inactive (or slight variation for multiple activities) is preferred.
- Update `/api/activities/summary` to return daily activity counts/booleans for the last 365 days instead of the weekly volume data.

## 2.4 Drag-and-Drop Activities on Monthly Plan
- Make the `DayColumn` component accept drag-and-drop actions.
- Allow users to click and hold an activity to drag it to another day.
- Use a lightweight drag-and-drop library like `@dnd-kit/core` and `@dnd-kit/sortable`, or standard HTML5 Drag and Drop API, to move activities between days.
- When an activity is dropped onto a new day, trigger a `PATCH /api/activities/:id` request to update its `activityDate` to the new day.
- Note: We won't allow dragging of "logged" activities or recurring templates, only standard scheduled activities.

### 2.5 UI Polish & Beautification
- **Add depth**: Change the light-mode `--background` CSS variable to `oklch(0.985 0.002 247)` (≈ zinc-50 / `#fafafa`), keeping `--card` at pure white. This creates visual layering between the page and card surfaces. Dark mode stays unchanged.
- **Soften borders**: Lighten `--border` in `:root` from `oklch(0.929 ...)` to `oklch(0.945 0.01 255)`. Increase `--radius` from `0.625rem` to `0.75rem`. Apply `border-border/50` as the default card border treatment.
- **Add subtle shadows**: Apply `shadow-sm` (or `shadow-[0_1px_3px_rgba(0,0,0,0.04)]`) to all `Card` components so they lift off the new off-white background.
- **Elevate stat cards**: Replace the current `bg-muted/30` stat blocks with white `Card` components featuring colored icon circles (`w-8 h-8 rounded-full bg-{color}/10` with full-color icon). Map: Dumbbell → blue, Timer → green, Flame → amber.
- **Body metric trend pills**: Wrap the trend indicator (TrendingUp/Down/Minus + delta value) in a `rounded-full px-2 py-0.5` pill badge. Color-code: green for improvement (weight ↓ or VO₂max ↑), red for regression, gray for flat.
- **Colored streak badges**: Replace the solid black `default` badge with color-coded variants by streak length: `<4` weeks → amber, `4–7` weeks → orange, `≥8` weeks → red/hot.
- **Soften role badges**: Increase the `RoleBadge` tinted background opacity from ~8% (`${color}15`) to ~20% (`${color}33`).

## 2.6 Training Phase Display on Monthly Plan
- Update `Scheduler` output (`ProposedActivity`) to include the actual `trainingPhase.description` or `limitationNotes` (or a truncated version) so it can be passed into the Activity notes.
- When an activity is auto-generated by a Training Phase, its notes should clearly state what to do during this phase (e.g., "Max Strength phase: Focus on low reps, high weight. [Limitation Note: Watch your shoulder]").
- Update `DayColumn` and the `ActivityForm` so that if an activity has notes (like the phase description), they are easily visible or accessible (e.g., a small tooltip or just visible when editing the activity, since the title already says "Week 1/4"). We will ensure the full phase description is saved into the Activity's `notes` column during schedule generation.

## 2.7 Training Phase Card Formatting
- The `TrainingPlanSection` component currently renders the `activePhase.description` inside a single `<p>` tag, which ignores the newline (`\n`) formatting generated by the backend.
- Update `TrainingPlanSection` to properly parse and render the phase description. The text is structured with uppercase headers (e.g., "ON-COURT FOCUS") followed by paragraphs.
- It should split the string by double newlines (`\n\n`) to get sections, then style the headers as bold/semi-bold smaller text, and the content as standard paragraph text, creating a clean, structured layout instead of a wall of text.

- Keep the Heatmap blocks simple. Use a base color (e.g., slate/gray for inactive, brand primary/green for active).
- Ensure the Goals modal handles responsive design (collapses to 1 column on mobile screens).
