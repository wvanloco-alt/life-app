# Implementation Plan: Calendar Management (7 Habits)

**Branch**: `001-calendar-management` | **Date**: 2026-02-27 | **Spec**: [spec.md](../../.specify/specs/001-calendar-management/spec.md)
**Input**: Feature specification from `.specify/specs/001-calendar-management/spec.md`

## Summary

Build a personal weekly planning and daily scheduling system based on Stephen Covey's "fourth generation" time management from *The 7 Habits of Highly Effective People*. The user defines life roles, writes a mission statement, sets weekly goals per role, classifies activities by urgency/importance quadrant, and tracks renewal activities (Sharpen the Saw). The app is a Next.js web application running locally, storing data in SQLite, with a modern UI built on shadcn/ui.

## Technical Context

**Language/Version**: TypeScript 5.x on Node.js 22.x
**Framework**: Next.js 15 (App Router, React Server Components)
**Primary Dependencies**: shadcn/ui (components), Tailwind CSS v4 (styling), Recharts (charts), date-fns (dates), Drizzle ORM (database), better-sqlite3 (SQLite driver)
**Storage**: SQLite (single file, local), auto-backup daily
**Testing**: Vitest (unit), React Testing Library (component)
**Target Platform**: Windows (local development), browser-based UI (localhost:3000)
**Project Type**: Web application (full-stack, single project)
**Performance Goals**: < 2s page load, < 500ms data operations
**Constraints**: Offline-capable (no external services), single-user, no authentication
**Scale/Scope**: 1 user, ~7 screens (see below), ~2000 rows/year

## Implementation Phases

### Phase A: Foundation (Screens 1-2)
Set up the Next.js project, database, and basic CRUD for Roles and Mission Statement. This establishes the project structure, proves the tech stack works, and gives the user their first interactive screens.

### Phase B: Core Planning (Screen 3)
Build the weekly planning session and weekly calendar view. This is the heart of the app -- where the user sets goals and schedules them.

### Phase C: Daily Use (Screens 4-5)
Build the daily adaptation view and quadrant matrix. This makes the app useful on a daily basis, not just during weekly planning.

### Phase D: Enrichment (Screens 6-7)
Add Sharpen the Saw tracking and analytics. These enhance the planning system with self-renewal and trend visibility.
